Welcome

Hello there! If you've come to this page then it would seem like you are interested in learning about a new and exciting browser standard - Cross-Origin Resource Sharing (CORS). If you've accidentally stumbled here, stay a little longer and you may learn a thing or two.

About Cross-Origin Resource Sharing

Although there is much more information in the other pages, I'll attempt to explain CORS in a nutshell. CORS provides a way for developers to safely and easily make asynchronous cross-domain requests. The CORS specification is ideal for web applications that serve public content or other web applications who currently use insecure methods such as JSONP or proxy servers. And because CORS is supported and implemented in all modern browsers, the experience is consistent and easy to add to your existing application.

Interested yet? If so get started by checking out our XSS page to learn the purpose of the Same-Origin Policy. The CORS page will give you a high-level overview of how the standard works and explains generic implementation. The tutorial page is a walkthrough of implementing CORS client-side using jQuery and server-side using Google App Engine with Python.
